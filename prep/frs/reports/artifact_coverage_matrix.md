# FRS Artifact Coverage Matrix

## Purpose

This document maps the current FRS runtime artifacts to likely source documents
and source sections.

It is an artifact-level coverage matrix, not a field-by-field lineage table.
Its purpose is to answer:

```text
For each current runtime artifact under plans/frs, what is the likely source
basis, how directly is it published, and what kind of prep work will be needed?
```

## Status Legend

- `direct`: published in the PDFs in a form close to the runtime artifact
- `derived`: source is published, but transformation or reshaping is required
- `referenced_not_published`: source basis is identified, but the full canonical values are not published in the PDFs
- `runtime_only`: runtime/modeling artifact rather than document-sourced data
- `computed`: generated by procedure rather than sourced from documents

## Main Coverage Conclusions

- FRS is strong on plan rules, class structure, and published decrement logic.
- The active-demographics artifacts look reproducible from the AV, but only after structured table extraction and normalization.
- The retiree-distribution artifact is likely reproducible only as a derived build output rather than a directly extracted table.
- Funding artifacts are source-grounded but far more normalized than the source presentation.
- Mortality remains the clearest case where the PDFs identify the basis but do not supply the full canonical runtime table values.
