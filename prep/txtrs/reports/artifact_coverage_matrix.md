# TXTRS Artifact Coverage Matrix

## Purpose

This document maps the current TXTRS runtime artifacts to likely source
documents and source sections.

It is an artifact-level coverage matrix, not a field-by-field lineage table.
Its purpose is to answer:

```text
For each current runtime artifact under plans/txtrs, what is the likely source
basis, how directly is it published, and what kind of prep work will be needed?
```

## Status Legend

- `direct`: published in the PDFs in a form close to the runtime artifact
- `derived`: source is published, but transformation or reshaping is required
- `referenced_not_published`: source basis is identified, but the full canonical values are not published in the PDFs
- `runtime_only`: runtime/modeling artifact rather than document-sourced data
- `computed`: generated by procedure rather than sourced from documents

## Main Coverage Conclusions

- TXTRS is strong on plan rules, cohort structure, and reduction logic.
- The main likely difficulty is not benefits but demographics: the current runtime grids appear more detailed and normalized than the published source tables.
- Entrant profile is no longer a pure source gap, but it still needs a reviewed build rule from the valuation’s banded source table to the runtime’s single-age form.
- Mortality is again a basis-known, values-not-fully-published case.
